# AI Tooling Usage Report

## Tools Used

- **Qwen / Cursor**: Generated boilerplate code for FastAPI, Streamlit, ingestion pipeline, and evaluation scripts.
- **ChatGPT**: Helped debug import errors, suggest prompt improvements, and draft documentation.
- **GitHub Copilot**: Autocompleted repetitive patterns (JSON structures, test cases, config files).

## What Worked Well

- Rapid prototyping: Went from zero to working RAG in <2 days.
- Debugging assistance: Fixed module resolution, API key issues, and path problems quickly.
- Documentation drafting: Generated first drafts of README, design doc, and tooling report.
- Code refactoring: Suggested cleaner function names, error handling, and structure.

## What Didn’t Work

- Sometimes generated outdated LangChain imports (required manual fixes).
- Occasionally over-engineered simple functions (trimmed for clarity).
- Rarely hallucinated non-existent parameters (verified against official docs).

## Overall Impact

AI tools accelerated development by ~70%, allowing focus on architecture, evaluation, and polish — critical for achieving Score 5. All final code was reviewed and validated manually.

## Ethical Note

All synthetic policy data was generated responsibly. No real employee data was used. Citations always trace back to original source documents.
